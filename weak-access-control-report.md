# Ethereum Weak Access Control Risk Report

_Generated: 2026-05-24_

This report summarizes Ethereum contracts where externally controlled privileged roles can still affect assets, upgrade authority, bridge trust, minting, withdrawals, or protocol liveness. The findings are framed as authority risk: they do not imply that value has moved or that any holder has acted improperly.

## Methodology

- BigQuery/log sweep: **1,348,790** unique contracts in the raw merged authority-candidate universe.
- Time range: local sweep artifacts contain authority-event timestamps from **2016-2026**.
- Ownership and authority patterns checked: `Ownable / Ownable2Step`, `owner()` roles held by EOAs, Safes, or contracts, `OpenZeppelin AccessControl` role memberships, `DEFAULT_ADMIN_ROLE` and role-admin hierarchies, `ProxyAdmin` ownership, `EIP-1967` admin slots, `Transparent proxy` admin authority, `UUPS` upgrade authority, and `Beacon` / beacon-proxy upgrade authority.

- Tenderly validation: all publication rows have drained-status simulation evidence. Bundled execution classification: **142 Yes** / **4 No** in the `Multicall` column.

## Scope

- Raw merged sweep universe: **1,348,790** unique contracts with authority-candidate events across the local sweep artifacts.
- Tenderly-verified drain paths in scope: **138** contracts.
- Value in scope: **$7.22B**

## Key findings

- **47** in-scope contracts are controlled by an EOA or a 2-of-N Safe; their value is **$339.1M**.
- The largest exposures in this snapshot are: USDT0 - LayerZero ($3.61B), Ethena ($1.85B), Ethena - LayerZero ($639.3M).
- The dominant risk pattern is active privileged authority over upgrade, bridge, minting, custody, withdrawal, or emergency-control paths.
- The report should be read as an exposure inventory, not as evidence of compromise or improper operator behavior.

## Recommended remediation

- Move single-key privileged roles to a multisig with independent signers and documented operational ownership.
- Raise low-threshold Safes where the controlled value or blast radius exceeds the intended trust model.
- Add timelocks, pause delays, or two-step execution for upgrade, bridge-peer, minting, custody, and rescue paths.
- Revoke stale roles and separate emergency, upgrade, treasury, bridge, and minting authority where possible.
- Monitor privileged role changes, proxy-admin changes, Safe owner changes, and bridge configuration updates on-chain.

- Public rows include protocol identity, contract, holder, role, control surface, nested-Safe status, multicall status, and reviewed function path.
- Exact calldata, helper bytecode, private Tenderly traces, and execution instructions are intentionally omitted.
- `Money at risk` is a point-in-time estimate from local artifacts and is not additive across duplicate rows for the same contract.
- Rows marked false-positive, config-only, skipped, reverted, value-moved-only, or manual-review-only are excluded from the publication table.
- Protocol labels were refreshed against public labels and verified-source evidence where available; unlabeled or generic names should be treated as best-effort attribution.

## Reading the table

- Each row is one active authority relationship. A contract can appear more than once when multiple roles or holders control the same value pot.
- `Money at risk` is the contract-level value estimate, so it is not additive across duplicate contract rows.
- `Control` is the signing surface that can exercise the authority: `EOA`, `Other`, or the Safe threshold such as `3/5`.
- `Multicall` means the validated path can be executed in one transaction or bundled transaction; `No` means it needs a delay, multiple transactions, or is not classified as bundleable.
